# 环境与地形选择策略

---

## 1. 赛题地形总览

### Standard 模式（5 种地形）

| 地形 | 难度特征 | 关键挑战 | 建议训练比例 |
|------|----------|----------|-------------|
| pyramid_slope | 上坡 | 前倾、打滑 | 0.20 |
| pyramid_slope_inv | 下坡 | 后倾、失速 | 0.20 |
| pyramid_stairs | 上楼梯 | 需抬腿高度 | 0.25 |
| pyramid_stairs_inv | 下楼梯 | 冲击、姿态失稳 | 0.25 |
| maze | 迷宫（0.5m 障碍） | 导航避障、转向 | 0.10 |

### Track 模式（5 段串联）

```
start → slope → slope_inv → stairs → stairs_inv → open_entry_maze → end
```

赛道终点固定为迷宫，且必须完整走完所有子地形。

---

## 2. 训练模式选择策略

### 2.1 推荐训练顺序

```
┌──────────────────────────────────────────────┐
│ Step 1: Standard 模式训练基础 locomotion       │
│   → 重点: slope + stairs 的稳定通过           │
│   → 时间: 总训练时间的 50%                    │
├──────────────────────────────────────────────┤
│ Step 2: Standard 模式包含 maze                 │
│   → 重点: 加入 maze 比例，训练转向/避障基础   │
│   → 时间: 总训练时间的 20%                    │
├──────────────────────────────────────────────┤
│ Step 3: Track 模式训练导航                     │
│   → 重点: 目标点跟踪、赛道穿越               │
│   → 时间: 总训练时间的 30%                    │
└──────────────────────────────────────────────┘
```

### 2.2 不推荐做法

- ❌ 从一开始就训练 Track 模式（基础运动能力不足会导致大量摔倒）
- ❌ maze 比例过高（迷宫奖励信号稀疏，会拖慢整体学习）
- ❌ 忽略 Standard 模式（Standard 和 Track 可能分别计分）

## 3. Standard 模式地形配比策略

### 3.1 初期训练（强调基础通过能力）

```toml
[terrain.standard.pyramid_slope]
proportion = 0.25

[terrain.standard.pyramid_slope_inv]
proportion = 0.25

[terrain.standard.pyramid_stairs]
proportion = 0.25

[terrain.standard.pyramid_stairs_inv]
proportion = 0.25

[terrain.standard.maze]
proportion = 0.0  # 初期不训练迷宫
```

理由：先让模型学会在坡面和楼梯上稳定行走，再引入迷宫。

### 3.2 中期训练（引入迷宫）

```toml
[terrain.standard.pyramid_slope]
proportion = 0.20

[terrain.standard.pyramid_slope_inv]
proportion = 0.20

[terrain.standard.pyramid_stairs]
proportion = 0.25

[terrain.standard.pyramid_stairs_inv]
proportion = 0.25

[terrain.standard.maze]
proportion = 0.10
```

### 3.3 后期精调（按评估弱项加权）

如果监控发现某地形分数明显低，增加该地形比例：

```
如 pyramid_stairs_inv 得分最低：
→ 将其 proportion 提高到 0.35
→ 其他按比例压缩
```

### 3.4 冠军方案的比例参考

冠军 go2_config.py 中的地形比例：
```python
terrain_proportions = [0.1, 0.1, 0.35, 0.25, 0.2]
# 对应 slope_up, slope_down, stairs_up, stairs_down, discrete
```

可以看出冠军**楼梯类比例最高**（0.35+0.25=0.60），说明楼梯是最大挑战。

## 4. 难度课程策略

### 4.1 terrain.num_rows（难度档数）

```
num_rows = 10  → 0~9 共 10 个难度等级
```

### 4.2 max_init_terrain_level（初始最大难度）

| 训练阶段 | max_init_terrain_level | 说明 |
|----------|----------------------|------|
| 初期 | 3 | 只在简单地形上训练 |
| 中期 | 5 | 逐步解锁中等难度 |
| 后期 | 9 | 全难度覆盖 |

### 4.3 difficulty_range

```toml
difficulty_range = [0.0, 1.0]  # 保持全范围，让 curriculum 自动调节
```

### 4.4 课程机制

平台内置 terrain curriculum：
- 机器人在当前难度表现好 → 下次 reset 到更高难度
- 表现差 → 降回低难度

**关键**：确保 reward 设计能让好的表现被正确识别（走得远 + 不摔倒）。

## 5. Track 模式环境配置

### 5.1 基础 Track 配置

```toml
[terrain]
mode = "track"
num_rows = 10
num_cols = 10
curriculum = true
max_init_terrain_level = 3

[terrain.track]
track_length = 5
sub_terrains = ["pyramid_slope", "pyramid_slope_inv", "pyramid_stairs", "pyramid_stairs_inv", "open_entry_maze"]
```

### 5.2 Track 模式训练要点

- **episode_length_s 要足够长**：赛道有 5 段，每段 8m，需要给足时间
  ```toml
  episode_length_s = 40  # 比 Standard 更长
  ```

- **命令范围要对齐评估**：
  ```toml
  [commands.limit]
  lin_vel_x = [0.0, 2.0]   # 评估可能限制为 [0, 1.0]，但训练可以更宽
  lin_vel_y = [-1.0, 1.0]
  ang_vel_z = [-1.5, 1.5]
  ```

- **open_entry_maze 必须放最后**（平台硬性要求）

### 5.3 Track 导航观测

需要在 `policy_observation_process.py` 中构造：

```python
def process(self):
    obs = self.default_observation()  # 301

    # 目标点相对位置（本体坐标系）
    goal_world = self.env.goal_positions  # (N, 3)
    robot_pos = self.env.scene["robot"].data.root_pos_w  # 需确认接口
    goal_rel = goal_world[:, :2] - robot_pos[:, :2]

    # 转到机器人本体坐标
    yaw = ...  # 从 projected_gravity 或其他方式获取
    cos_yaw, sin_yaw = torch.cos(yaw), torch.sin(yaw)
    goal_local_x = cos_yaw * goal_rel[:, 0] + sin_yaw * goal_rel[:, 1]
    goal_local_y = -sin_yaw * goal_rel[:, 0] + cos_yaw * goal_rel[:, 1]

    # 距离和方向
    dist = torch.norm(goal_rel, dim=1, keepdim=True)
    goal_features = torch.cat([goal_local_x.unsqueeze(1),
                               goal_local_y.unsqueeze(1),
                               dist], dim=1)  # 3 维

    obs = torch.cat([obs, goal_features], dim=-1)  # 304 维
    return obs
```

## 6. 域随机化与评估差异

### 6.1 关键差异

| 配置 | 训练时 | 评估时 |
|------|--------|--------|
| domain_rand | ✅ 开启 | ❌ 关闭 |
| noise | ✅ 开启 | ❌ 关闭 |
| push_robots | ✅ 开启 | ❌ 关闭 |
| curriculum | ✅ 开启 | ❌ 关闭 |

### 6.2 应对策略

**训练后期降低随机化强度**：

```
训练前 70%: 强随机化（friction=[0.2, 2.5], push=1.5 m/s）
训练后 30%: 弱随机化（friction=[0.5, 1.2], push=0.3 m/s）
```

这样模型既有泛化能力，又能在"干净"环境中表现流畅。

**或者**：保存多个 checkpoint，用评估任务挑选在无噪声下表现最好的模型。

## 8. 环境参数对比：冠军 vs 本届平台

| 参数 | 冠军方案 | 本届平台 | 差异影响 |
|------|----------|----------|----------|
| num_envs | 8192 | ≤4096 | batch 变小，需调 lr |
| episode_length_s | ~20s | 25s (std) / 40s (track) | 平台更长 |
| terrain types | 5 类 (Isaac Gym) | 5 类 (Isaac Lab) | 基本对齐 |
| height_scan | 187 点 | 256 点 (16×16) | 平台更细 |
| 域随机化 | 10+ 项 | 3 项 | 差距大 |
| PD control | 可自定义 | 平台固定 | 无法迁移 |
| action_scale | 0.25 | 0.25 | 一致 |
| decimation | 4 | 平台内部 | 无法调 |

## 9. 总结：环境选择决策矩阵

```
Q: 我应该先训练哪种模式？
A: Standard 模式（打基础）

Q: maze 要不要训？
A: 要！但占比 10~15%，别太高

Q: Track 什么时候开始？
A: Standard 各地形通过率 > 70% 后再切 Track

Q: num_envs 设多少？
A: 4096（训练快）；评估用 16（看视频）

Q: 难度从几开始？
A: max_init_terrain_level = 3，让 curriculum 自动升

Q: episode_length 设多少？
A: Standard 20~25s；Track 35~40s
```
