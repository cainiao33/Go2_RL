# 训练策略规划

---

## 1. 总体策略：对齐评分公式

### Standard 模式评分驱动

```
总分 = 0.4×距离 + 0.2×时间 + 0.2×能耗 + 0.2×姿态
```

**训练优先级**：
1. **距离 (0.4)**：必须走穿地形（≥3.9m）→ 速度跟踪 + 前进引导
2. **姿态 (0.2)**：roll/pitch 偏移小 → flat_orientation 惩罚加大
3. **能耗 (0.2)**：关节功率低 → joint_torques + action_smoothness
4. **时间 (0.2)**：仅走穿才得分 → 走穿后用时越短越好

### Track 模式评分驱动

```
总分 = 完成率 × (0.4×时间 + 0.4×姿态 + 0.2×能耗)
```

**训练优先级**：
1. **完成率（乘数）**：不完成 = 0 分 → 导航必须可靠
2. **时间 (0.4)**：速度要快
3. **姿态 (0.4)**：稳定性很重要
4. **能耗 (0.2)**：次要

## 2. Reward 工程策略

### 2.1 Standard 模式 reward 方案

#### 第一层：基础 locomotion

| 奖励 | 权重 | 目的 |
|------|------|------|
| track_lin_vel_xy | 1.5 | 跟踪速度命令（核心驱动力）|
| track_ang_vel_z | 0.5 | 跟踪转向命令 |
| lin_vel_z | -2.0 | 抑制弹跳 |
| ang_vel_xy | -0.1 | 抑制 roll/pitch 摆动 |
| flat_orientation | -2.0 | **加大**平坦姿态权重（姿态分占 0.2）|

#### 第二层：安全与约束

| 奖励 | 权重 | 目的 |
|------|------|------|
| undesired_contacts | -1.5 | 防摔倒（摔倒 = 终止 = 0 距离分）|
| dof_pos_limits | -5.0 | **加大**关节极限惩罚 |
| action_rate | -0.01 | 动作平滑 |
| action_smoothness | -0.02 | 二阶平滑（借鉴冠军）|

#### 第三层：能效优化

| 奖励 | 权重 | 目的 |
|------|------|------|
| joint_torques | -2e-4 | 抑制大力矩（**加大**，对齐能耗分）|
| joint_acc | -2.5e-7 | 抑制加速度突变 |

#### 第四层：步态质量（新增）

| 奖励 | 权重 | 目的 |
|------|------|------|
| feet_air_time | 1.0 | 鼓励长步幅、正常步态 |
| stand_still | -0.5 | 零命令时不乱动 |

### 2.2 Track 模式 reward 方案

在 Standard 基础上新增：

| 奖励 | 权重 | 目的 |
|------|------|------|
| reach_goal | **30.0** | 完成奖励大幅提高（完成率是乘数）|
| goal_direction | 3.0 | 朝目标方向移动 |
| navigation_progress | 5.0 | 距离缩短 |
| forward_velocity | 1.0 | 沿赛道方向快速前进 |

### 2.3 Reward 调参经验（来自冠军）

- **动态 sigma**：速度跟踪奖励的 σ 随地形难度和命令大小动态调整
  - 简单地形 + 低速 → 小 σ（精确跟踪）
  - 困难地形 + 高速 → 大 σ（容许偏差）
- **奖励课程**：某些 reward 权重随训练步数渐增
  - 前期重点：存活 + 速度跟踪
  - 后期重点：能效 + 姿态 + 平滑

## 3. 课程学习策略

### 3.1 地形课程（平台支持）

```toml
[terrain]
curriculum = true
max_init_terrain_level = 3    # 初始只到 level 3
difficulty_range = [0.0, 1.0] # 逐步解锁全难度
```

平台机制：表现好的机器人会被升级到更难的地形。

### 3.2 速度命令课程

**阶段 1（前 30% 训练时间）**：
```toml
[commands.ranges]
lin_vel_x = [0.0, 0.8]
lin_vel_y = [-0.3, 0.3]
ang_vel_yaw = [-0.5, 0.5]
```

**阶段 2（中 40%）**：
```toml
[commands.ranges]
lin_vel_x = [-0.5, 1.5]
lin_vel_y = [-0.5, 0.5]
ang_vel_yaw = [-1.0, 1.0]
```

**阶段 3（后 30%）**：
```toml
[commands.ranges]
lin_vel_x = [-1.0, 2.0]
lin_vel_y = [-1.0, 1.0]
ang_vel_yaw = [-1.5, 1.5]
```

> ⚠️ 平台不支持运行时动态修改 TOML。需要**分多次训练任务**（preload_model 继续训练）或在 reward 中实现 soft curriculum。

### 3.3 Soft Curriculum 实现（推荐）

在 `reward_process.py` 中根据 episode 步数/训练迭代实现 curriculum：

```python
def _reward_track_lin_vel_xy(self):
    """带动态 sigma 的速度跟踪"""
    # 根据当前地形难度动态调整 sigma
    base_sigma = 0.25
    # 如果有 terrain level 信息可用：
    # difficulty_factor = self._get_terrain_difficulty()
    # sigma = base_sigma + 0.1 * difficulty_factor
    sigma = base_sigma
    error = torch.sum(torch.square(self.commands[:, :2] - self.base_lin_vel[:, :2]), dim=1)
    return torch.exp(-error / sigma)
```

## 4. 域随机化策略

### 4.1 平台可用的域随机化

| 参数 | 默认值 | 建议值 | 理由 |
|------|--------|--------|------|
| friction_range | [0.3, 1.5] | [0.2, 2.5] | 更宽范围提升泛化 |
| push_robots | true | true | 保持 |
| push_interval_s | 15 | 8 | 更频繁推力提升鲁棒性 |
| max_push_vel_xy | 0.5 | 1.5 | 更强推力 |
| add_noise | true | true | 观测噪声保持 |

### 4.2 通过 reward 补偿不可用的域随机化

冠军方案有但平台没有的：
- motor strength randomization → 加大 `joint_torques` 惩罚权重
- action delay → 加大 `action_smoothness` 权重
- PD gain randomization → 加大 `action_rate` 权重
- base COM randomization → 加大 `flat_orientation` 权重

## 5. 分阶段训练计划

### Phase 1: Standard Locomotion（建议 8000~15000 iterations）

```
目标：全 5 种地形通过率 > 80%
配置：standard 模式, num_envs=4096, curriculum=true
重点：reward 工程 + 地形课程
验证：在各地形上的 distance_score 和 completed_count
```

### Phase 2: Track Navigation（建议 5000~10000 iterations）

```
前置：Phase 1 的模型作为预训练
目标：Track 完成率 > 60%
配置：track 模式, 加入 goal obs, 加入导航 reward
重点：goal_direction + reach_goal + navigation_progress
验证：completed_count / total, time_score
```

### Phase 3: 精调与鲁棒性（建议 3000~5000 iterations）

```
前置：Phase 2 的模型
目标：提升能耗分和姿态分
配置：加大 energy/posture reward 权重
重点：降低域随机化强度，让模型收敛到更"优雅"的步态
验证：energy_score, pose_score 提升
```

## 7. 关键监控指标

### 训练健康度

- `total_loss` 持续下降
- `entropy_loss` 缓慢下降（过快 = 探索不足）
- `reward_mean` 持续上升

### Standard 表现

- `distance_score` > 0.8（各地形）
- `completed_count` >> `abnormal_count`
- `pose_score` > 0.7
- `energy_score` > 0.6

### Track 表现

- `completed_count / (completed + abnormal + timeout)` > 0.6
- `time_score` > 0.5
- `pose_score` > 0.7
