## 1. 模型架构适配

### 1.1 当前 baseline（比赛示例）

```
Actor:  301 → [512, 256, 128] → 12   (ELU)
Critic: 316 → [512, 256, 128] → 1    (LayerNorm + ELU)
```

### 1.2 冠军方案模型

```
Teacher Actor:  263 (privileged) → [512, 256, 128] → 12
Student Actor:  45 × history_length (flattened) → [512, 256, 128] → 12
Critic:         263 (privileged) → [512, 256, 128] → 1
```

### 1.3 适配方案

**推荐保留当前 baseline 的 301 维 policy obs 输入**，因为平台已经给了 height_scan（256 维），比冠军方案的 student（blind 45 维）信息更丰富。

改造点：
- **隐藏层可加宽**：从 `[512, 256, 128]` 改为 `[512, 512, 256]`，提升表达能力
- **Critic 可加深**：Critic 有 316 维特权信息，网络可以更大
- **Track 模式**：在 `policy_observation_process.py` 中拼接目标点特征（+6~8 维），对应修改 Actor 输入维度

```python
# policy_observation_process.py 改造示例
def process(self):
    obs = self.default_observation()  # 301 维
    if hasattr(self.env, 'goal_positions') and self.env.goal_positions is not None:
        # 构造相对目标方向和距离
        goal_rel = self._compute_goal_relative()  # 例如 [dx, dy, dist, cos_yaw, sin_yaw]
        obs = torch.cat([obs, goal_rel], dim=-1)  # 301 + 5 = 306 维
    return obs
```

同步修改：
- `StageConfig.num_obs` 或新增 `TrackConfig`
- `critic_observation_process.py` 中也加入对应特征
- `actor_critic.py` 的 `mlp_input_dim_a` 自动从 config 读取

## 3. 训练工作流适配

### 3.1 当前 workflow 结构（保持不变）

```
workflow() → 数据收集(run_episodes_) → agent.learn() → 模型保存
```

### 3.2 改造点

**A. 多阶段训练支持**

在 `conf.py` 中新增 `TrackConfig`：

```python
class TrackConfig(StageConfig):
    """Track 模式训练阶段"""
    stage_name = "track"
    toml_name = "train_env_conf_track.toml"
    num_obs = 306  # 301 + 5 (goal features)
    num_critic_observations = 321  # 316 + 5
    # 其他参数...
```

**B. 预训练加载**

Standard 阶段训练好 locomotion → 保存模型 → Track 阶段 `preload_model=true` 继续训练导航。

**C. 课程学习集成**

在 workflow 中根据监控指标动态调整 TOML 配置（如有接口支持），或在 reward 中实现 soft curriculum：

```python
# 根据训练步数逐步增加速度命令范围
progress = min(1.0, self.common_step_counter / total_steps)
cmd_scale = 0.5 + 0.5 * progress  # 从 50% → 100% 命令范围
```

## 5. PPO 超参适配

### 5.1 冠军方案 vs 当前 baseline

| 参数 | 冠军 | baseline | 建议 |
|------|------|----------|------|
| num_envs | 8192 | 2048 | 4096（平台上限） |
| num_steps_per_env | 24 | 48 | 24~32（更频繁更新） |
| num_learning_epochs | 5 | 5 | 5 |
| num_mini_batches | 4 | 4 | 4 |
| lr | 1e-3 | 3e-4 | 1e-3 → adaptive |
| clip_param | 0.2 | 0.2 | 0.2 |
| gamma | 0.99 | 0.99 | 0.99 |
| lam | 0.95 | 0.95 | 0.95 |
| entropy_coef | 0.01 | 0.01 | 0.01 |
| desired_kl | 0.01 | 0.01 | 0.01 |
| max_grad_norm | 1.0 | 1.0 | 1.0 |

### 5.2 关键调整

- **batch_size** = num_envs × num_steps_per_env = 4096 × 24 = 98304
- **mini_batch_size** = 98304 / 4 = 24576
- **学习率**：初始 1e-3，通过 adaptive KL 自动衰减
- **num_steps_per_env 降低**：更频繁更新有利于 on-policy 效率

### 6.2 评估时注意

- 评估关闭域随机化和噪声
- 评估 command limit 可能与训练不同（如 track 模式评估 `lin_vel_x = [0, 1.0]`）
- 需确保模型在 **无噪声、无推力** 下仍然稳定

**建议**：训练后期降低噪声和推力强度，或增加无噪声 episode 的比例。
